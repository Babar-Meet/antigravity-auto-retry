A cross-platform GUI automation bot that continuously monitors the screen for a "Retry" button image and automatically clicks it when found.

## Purpose

This project eliminates the need for manual intervention when an application, game, or service displays a retry prompt. It uses screen capture and template matching to detect the target button and simulates a mouse click on it, running indefinitely until the operator stops it.

Intended for anyone who encounters recurring retry prompts during workflows such as software installations, game reconnect screens, or any GUI-based operation that requires periodic retry clicking.

## Features

- **Image-based detection** – Locates the Retry button on screen using OpenCV template matching with configurable confidence threshold.
- **Automatic clicking** – Performs a mouse click as soon as the button is detected.
- **Configurable polling interval** – Checks every 9 seconds by default; adjustable via the `CHECK_INTERVAL_SECONDS` constant.
- **Cross-platform support** – Ships with setup and run scripts for Windows (`.bat`), Linux (`.sh`), and macOS (`.sh`).
- **Graceful error handling** – Catches `pyautogui.ImageNotFoundException` and general exceptions without crashing.
- **Timestamps in log output** – Every detection cycle and action is printed with an `HH:MM:SS` timestamp for observability.
- **Countdown timer** – Displays a per-second countdown between checks for user feedback.

### Request / Data Flow

There are no network requests. The data flow is entirely local:

1. **Screen capture** – PyAutoGUI (via OpenCV's `matchTemplate`) captures the current screen.
2. **Template matching** – The captured image is compared against `images/retrybutton.png` using normalized cross-correlation. If the best match exceeds the confidence threshold (0.7 / 70 %), the match location is returned.
3. **Click execution** – `pyautogui.click()` moves the cursor to the matched center coordinate and performs a left mouse click.
4. **Wait** – A second-by-second countdown loop pauses execution for the configured interval, then repeats from step 1.

### Error Handling

- `pyautogui.ImageNotFoundException` – Raised when the `confidence` parameter is used and no match passes the threshold. Caught and logged, loop continues.
- Generic `Exception` – Any other runtime error is caught, logged, and the loop continues. The bot does not stop on errors.

### State Management

The bot maintains no persistent state. Each iteration is stateless and independent.

## Configuration

There is no external configuration file. All tuning is done by editing constants at the top of `bot.py`:

- `RETRY_BUTTON_IMG` (`bot.py:5`) – Path to the template image. Change if the image is renamed or moved.
- `CHECK_INTERVAL_SECONDS` (`bot.py:8`) – Polling interval in seconds. Lower values increase responsiveness but consume more CPU.
- `confidence=0.7` (`bot.py:17`) – OpenCV template matching confidence threshold. Accepted range is `0.0`–`1.0` where `1.0` requires an exact pixel match. Lower values tolerate screen resolution differences, color shifts, and small UI changes but increase false-positive risk.

## How It Works

1. The bot loads `images/retrybutton.png` into memory as a template.
2. Every 9 seconds, it captures the current screen and uses OpenCV template matching (via PyAutoGUI) to find a region that matches the template with at least 70 % confidence.
3. If a match is found, the bot moves the mouse cursor to the center of the matching region and clicks the left mouse button.
4. If no match is found, it logs the result and waits for the next cycle.
5. The loop continues indefinitely until the user presses Ctrl+C.

Because the bot works entirely through screen capture and mouse simulation, it is oblivious to the application it is clicking — it will click any Retry button that visually matches the reference image, regardless of which window owns it.
